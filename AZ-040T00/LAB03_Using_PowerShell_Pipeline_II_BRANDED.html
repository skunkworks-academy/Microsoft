PowerShell Pipeline II

AZ-040: Automating Administration with PowerShell

Key Topics Learned

  • Chaining cmdlets using | to streamline administration tasks.
  • Filtering and sorting objects directly in the pipeline.
  • Using ForEach-Object and Where-Object efficiently.
  • Understanding object types passed through the pipeline.
  • Error handling and conditional processing with the pipeline.

Quick Reference Commands

  • Get-Process | Sort-Object CPU -Descending | Select-Object -First 5
  • Get-Service | Where-Object {$_.Status -eq "Running"}
  • Get-Content .\log.txt | ForEach-Object { $_.ToUpper() }
  • Get-ADUser -Filter * | Select-Object Name, Enabled

Troubleshooting Tips

  • Use Get-Member to inspect object properties in the pipeline.
  • Check for null or unexpected data with Where-Object { $_ }.
  • Use -ErrorAction Stop to capture errors in scripts.
  • Pipe to Out-GridView for visual data inspection.

Best Practices

  • Always test pipeline commands with -WhatIf where available.
  • Combine filtering and sorting early in the pipeline for better performance.
  • Document reusable pipeline snippets in scripts for consistency.
  • Use Try...Catch blocks for robust error handling.

One-pager: Configuring Windows PowerShell & Finding/Running Commands
2025-08-22
Objective

Open and configure PowerShell console & ISE, discover commands efficiently, and use Help/About topics to learn core concepts.

Get-HelpUpdate-HelpGet-CommandGet-MemberStart-TranscriptGet/Set-Alias

Quick Reference (copy & paste)

# Help system
Update-Help -Force                       # run elevated or with -SourcePath for offline
Get-Help Get-Process -Examples           # or -Online, -Full

# Find commands fast
Get-Command -Verb Get -Noun *Service*    # discover by verb/noun wildcards
Get-Command -Module *Net*                # search within module names
Get-Command -Name *Firewall* -CommandType Cmdlet,Function

# Inspect objects
Get-Service | Get-Member                 # inspect properties & methods
Get-Service | Where-Object Status -eq 'Running' | Select Name,Status

# Aliases (prefer full names in scripts)
Get-Alias | Sort-Object Definition | Select Name,Definition
Set-Alias ll Get-ChildItem

# Transcript
Start-Transcript C:\DayOne.txt
Stop-Transcript

Tip: Keep Format-* at the end of pipelines; format early and you lose rich objects for further processing.

What you did

  • Launched the 64-bit PowerShell console as Administrator and adjusted appearance (font, colors, layout).
  • Customized ISE view (single-pane, hide Command pane) and font size.
  • Used Get-Help/Get-Command to discover commands and ran basic admin tasks.
  • Started a shell transcript to capture commands and output.

Troubleshooting & Root Cause

SymptomLikely Cause → Fix
Update-Help fails / cannot download Proxy/TLS or permissions → run elevated, ensure TLS 1.2; set proxy for WinHTTP; or use offline store: Save-Help on a connected PC and Update-Help -SourcePath here.
Help content missing for modules Module not installed or not imported → Get-Module -ListAvailable, Import-Module, then retry Update-Help.
Transcript path errors Path invalid/locked → choose a writable path (e.g., $env:TEMP\DayOne.txt) or run as admin for system paths.
Command not found Wrong module or edition → check with Get-Command, then Install-Module or use Windows PowerShell if cmdlet isn’t in PS7.
Console shows (x86) 32‑bit host launched → start 64‑bit PowerShell (no (x86) in title) to access full provider paths.

Ops / Exam Tips

  • Prefer full cmdlet names in scripts; aliases are fine interactively.
  • Use splatting for readability: $p=@{Name='bits';Verbose=$true}; Get-Service @p.
  • Learn common verbs: Get-Verb (e.g., Get/Set/New/Remove/Test).

Try This (extra practice)

  • Find the command to resolve a DNS name, then run it for a host you choose.
  • List enabled Windows Firewall rules and filter to only enabled ones.
  • Open the newest 10 Security log entries with Get-WinEvent.
  • Search About topics: Get-Help about_* → read about comparison operators and environment variables.
